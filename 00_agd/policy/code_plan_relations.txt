# code-plan relation policy (repo-root relative)
# 코드 기획 연관 정책 (저장소 루트 상대 경로)
#
# parser note:
# 파서 참고:
# - blank lines and lines starting with # are ignored
# - only relation lines are parsed
# - 빈 줄과 # 주석 라인은 무시됩니다.
# - 실제 연관 라인만 파싱됩니다.
#
# relation line syntax:
# <left-doc.agd>#<SECTION-ID> <-> <right-doc.agd>#<SECTION-ID>
# 연관 라인 문법:
# <left-doc.agd>#<SECTION-ID> <-> <right-doc.agd>#<SECTION-ID>
#
# -----------------------------------------------------------------------------
# [WRITE RULES]
# [작성 규칙]
# -----------------------------------------------------------------------------
# 1) one relation per line (no comma-separated endpoints)
# 2) for multiple relations, repeat lines
#    - 1:N => repeat left endpoint
#    - N:1 => repeat right endpoint
# 3) use repo-root relative paths
# 4) both / and \ are allowed, but / is recommended for consistency
# 5) section id should be explicit and stable (ex: SYS-020, FP-010)
#
# 1) 한 줄에는 연관 1개만 작성합니다(콤마로 다중 endpoint 나열 금지)
# 2) 여러 연관은 줄을 반복해 작성합니다
#    - 1:N => 왼쪽 endpoint 반복
#    - N:1 => 오른쪽 endpoint 반복
# 3) 저장소 루트 기준 상대 경로를 사용합니다
# 4) /, \ 모두 허용하지만 일관성을 위해 / 권장
# 5) section id는 명시적이고 안정적인 값을 사용합니다 (예: SYS-020, FP-010)
#
# -----------------------------------------------------------------------------
# [ORDERING RULES]  (strongly recommended for large files)
# [정렬 규칙] (대용량 파일에서 강력 권장)
# -----------------------------------------------------------------------------
# sort by:
#   (a) left doc path
#   (b) left section id
#   (c) right doc path
#   (d) right section id
# 아래 순서로 정렬합니다:
#   (a) left 문서 경로
#   (b) left 섹션 ID
#   (c) right 문서 경로
#   (d) right 섹션 ID
